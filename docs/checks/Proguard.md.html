(#) Using obsolete ProGuard configuration !!! ERROR: Using obsolete ProGuard configuration This is an error, and is also enforced at build time when supported by the build system. For Android this means it will run during release builds. Id : `Proguard` Summary : Using obsolete ProGuard configuration Severity : Fatal Category : Correctness Platform : Android Vendor : Android Open Source Project Affects : Shrinking configuration files Editing : This check runs on the fly in the IDE editor See : https://issuetracker.google.com/36928077 Implementation : [Source Code](https://cs.android.com/android-studio/platform/tools/base/+/mirror-goog-studio-master-dev:lint/libs/lint-checks/src/main/java/com/android/tools/lint/checks/ProguardDetector.java) Tests : [Source Code](https://cs.android.com/android-studio/platform/tools/base/+/mirror-goog-studio-master-dev:lint/libs/lint-tests/src/test/java/com/android/tools/lint/checks/ProguardDetectorTest.kt) Using `-keepclasseswithmembernames` in a proguard config file is not correct; it can cause some symbols to be renamed which should not be. Earlier versions of ADT used to create proguard.cfg files with the wrong format. Instead of `-keepclasseswithmembernames` use `-keepclasseswithmembers`, since the old flags also implies “allow shrinking” which means symbols only referred to from XML and not Java (such as possibly CustomViews) can get deleted.
