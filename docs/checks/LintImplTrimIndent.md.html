There are advantages to **not** putting `.trimIndent()` in the code. For test files, if you call for example `kotlin(""\"source code"\""\")` then IntelliJ/Android Studio will syntax highlight the source code as Kotlin. The second you add in a .trimIndent() on the string, the syntax highlighting goes away. For test files you can instead call ".indented()" on the test file builder to get it to indent the string. !!!
