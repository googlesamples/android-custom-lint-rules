The lint detector also looks at the `id` of the view, and if the id offers a hint of the purpose of the field (for example, the `id` contains the phrase `phone` or `email`), then lint will also ensure that the `inputType` contains the corresponding type attributes. If you really want to keep the text field generic, you can suppress this warning by setting `inputType="text"`. !!! Tip This lint check has an associated quickfix available in the IDE.
