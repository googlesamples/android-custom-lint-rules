(#) Using mock location provider in production !!! ERROR: Using mock location provider in production This is an error, and is also enforced at build time when supported by the build system. For Android this means it will run during release builds. In Gradle projects, that means you should only request this permission in a test or debug source set specific manifest file. To fix this, create a new manifest file in the debug folder and move the `` element there. A typical path to a debug manifest override file in a Gradle project is src/debug/AndroidManifest.xml. !!!
